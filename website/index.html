Why this problem exists

A weak branch can break many workflows. Random and static baselines often never touch it.

Rare branches

Deep control flow in model outputs is only exposed when tests are steered toward structurally new states.

False confidence

Passing shallow checks can still hide behavioral or state-level failures.

Comparability

Different budgets, seeds, and checkers can invalidate method-to-method comparisons.

Core ideas

Behavioral exploration

Runs become compact episode trees; promising branches mutate under valid constraints.

Single fairness envelope

All methods share the same seed, budget, and execution envelope for fair comparison.

Evidence-first defects

Layered accounting avoids duplicate inflation and keeps findings auditable.

Artifact-first reproducibility

Seeded journals and manifests are kept for replay and reviewer inspection.

System architecture

Search and execution are separated to reduce feedback coupling.

1. Candidate builder

Inputs and prompt materialization

Same pipeline for all methods: seed -> candidate -> runner prompt.

2. Execution

Isolated skill runs

Deterministic constraints with fixed timeouts and captured artifacts.

3. Trace extraction

Episode segmentation

Logs are normalized into branching episodes.

4. Search policy

Tree + guard mutation

Priority is given to unexplored and high-potential branches.

5. Oracle

Checks and verification

Static checks, property checks, and generated checks are scored consistently.

6. Reporting

Yield and confidence

Results are grouped, confirmed, and exported with seed-aware metadata.

Method

1) Build candidate

Seed + policy generate one runnable candidate.

2) Execute and trace

Isolated execution captures artifacts, outputs, and state.

3) Compress behavior

Runs compress into normalized episode signatures.

4) Mutate target

Promising branches are mutated with validity checks.

5) Evaluate

Candidate replay goes through the fixed oracle and accounted checks.

The differentiator is branch policy and ranking in step 4.

Method difference (fair policy)

Random ignores execution feedback and always samples directly from the search space.

Toggle to see the behavior difference.

Failure model

1. Signal

Any check failure triggers a raw signal.

2. Grouping

Equivalent traces are merged by normalized signatures.

3. Confirmation

Repeats are re-run under frozen conditions.

4. Campaign score

Confirmed defects become campaign yield.
